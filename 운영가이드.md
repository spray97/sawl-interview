# 수원외고 면접후기 서비스 운영 가이드

수원외고 재학생(고3)에게 선배들의 대입 면접후기를 제공하는 검색 서비스.
GitHub Pages 기반 정적 웹사이트로, 별도 서버 없이 무료 운영.

- **서비스 URL**: https://spray97.github.io/sawl-interview
- **GitHub 저장소**: https://github.com/spray97/sawl-interview
- **데이터 현황**: 15기~18기 총 409건 (2026년 기준)

---

## 인증코드

학생들이 서비스 입장 시 사용하는 코드. 담임선생님 경유 배포 권장.

| 연도 | 인증코드   | 비고 |
|------|-----------|------|
| 2026 | (담임선생님께 문의) | 초기 설정값 — 별도 전달 |
| 2027 |            | 매년 초 교체 |

> 인증코드 평문은 이 문서에 기재하지 않습니다. 담임선생님 또는 전임 담당자에게 직접 문의하세요.
> 코드는 SHA-256 해시로 변환되어 `auth.json`에만 저장됩니다.

## 매년 해야 할 일 (연초)

### 1. 인증코드 교체

새 코드의 SHA-256 해시값 계산:

```bash
# Windows PowerShell
echo -n "sawl2027" | sha256sum

# 또는 온라인 도구: https://emn178.github.io/online-tools/sha256.html
# 입력값: sawl2027  →  해시값 복사
```

`auth.json` 수정:

```json
{
  "hash": "여기에_새_해시값_입력",
  "hint": "담임선생님께 문의하세요",
  "expires": "2027-12-31"
}
```

GitHub에 업로드: 저장소 → `auth.json` 클릭 → 연필(수정) 아이콘 → 값 변경 → **Commit changes**

### 2. 새 기수 데이터 추가 (선택)

새 기수(예: 19기) 면접후기 파일이 생기면 DB 파이프라인 재실행 후 `interviews.json` 교체.
→ 파이프라인 관련 코드: `D:\000_temp\000-clacostudy\hwpx-parsing\interview-db\`
→ 파이프라인 담당자에게 별도 문의

## GitHub 저장소 파일 구성

```

├── index.html        ← 전체 웹앱 (수정 불필요)
```

## 파일 수정 방법 (GitHub 웹에서)

1. https://github.com/spray97/sawl-interview 접속
2. 수정할 파일 클릭
3. 오른쪽 상단 연필 아이콘(Edit) 클릭
4. 내용 수정
5. 하단 **Commit changes** 클릭
6. 약 1분 후 서비스에 자동 반영

## 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 인증코드 입력해도 안 됨 | 해시값 불일치 | sha256 재계산 확인, 입력값 공백 주의 |
| 페이지가 열리지 않음 | Pages 비활성화 | Settings → Pages → Source 재설정 |
| 데이터가 예전 것 | 캐시 | Ctrl+Shift+R 강력 새로고침 |
| 로컬 테스트 필요 | - | `python -m http.server 8080` 후 localhost:8080 |

## DB 파이프라인 (데이터 갱신 필요 시)

면접후기 한글/PDF 파일 → SQLite DB → JSON 변환 파이프라인.

- **코드 위치**: `D:\000_temp\000-clacostudy\hwpx-parsing\interview-db\`

- **출력물**: `data/output/interviews.json` → `sawl-interview/interviews.json`으로 복사
